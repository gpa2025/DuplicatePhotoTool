# Versioning Policy

This project uses **Semantic Versioning 2.0.0**  
https://semver.org/

## Format

MAJOR.MINOR.PATCH

### MAJOR
- Breaking changes
- Significant redesigns
- Behavior changes requiring user action

### MINOR
- New features
- Backwards‑compatible improvements
- Enhancements to GUI, logging, caching, etc.

### PATCH
- Bug fixes
- Performance improvements
- Documentation updates
- Non‑breaking internal changes

## Examples

### 1.0.0 → 1.1.0
Added new selection mode or GUI feature.

### 1.1.0 → 2.0.0
Changed default behavior or modified output structure.

### 1.1.0 → 1.1.1
Fixed a bug in hashing or caching.
